# Модель, репозиторий и работа с базой данных

В ходе работы был создан класс модели `User`, который описывает структуру данных пользователя (идентификатор, имя и год рождения). Это позволило абстрагироваться от структуры таблицы базы данных и работать с объектами.

Для создания и управления базой данных был реализован класс `DatabaseHelper`, унаследованный от `SQLiteOpenHelper`. В нём была определена таблица пользователей, а также реализованы методы создания и обновления базы данных. При инициализации базы добавляется тестовая запись.

Основная логика работы с данными реализована в классе `DatabaseAdapter`, который выполняет роль репозитория. В данном классе реализованы методы:
* открытия и закрытия подключения к базе данных;
* добавления (insert);
* удаления (delete);
* обновления (update);
* получения одного пользователя по id (getUser);
* получения списка всех пользователей (getUsers).

Пользовательский интерфейс приложения включает `MainActivity`, в которой отображается список пользователей с помощью `ListView` и `UserActivity`, предназначенную для добавления, редактирования и удаления пользователей.

В `MainActivity` данные загружаются из базы данных при каждом возобновлении активности и отображаются через `ArrayAdapter`. При выборе элемента списка выполняется переход к экрану редактирования.

В `UserActivity` реализована логика сохранения и удаления данных. В зависимости от наличия идентификатора пользователя определяется режим работы: добавление нового пользователя или редактирование существующего.

     Была реализована работа с базой данных с использованием модели и паттерна «репозиторий», что делает работу с данными приложения более гибкой по сравнению с прямой работой через Cursor.
